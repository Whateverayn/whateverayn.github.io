Anacondaをインストールした後, ターミナルを開くたびにプロンプトの先頭に(base)と表示され, base環境が自動で有効化されるようになる. この自動有効化をオフにする方法と, 必要な時に手動で(base)環境を有効/無効にする方法について残す.

ターミナル起動時の(base)環境自動有効化をオフにする

Anacondaをインストールした際, デフォルトで設定されるこの挙動は, conda configコマンドで変更できる. 

conda config --set auto_activate_base false

このコマンドを実行後, 新しいターミナルを開くと, プロンプトの先頭に(base)が表示されなくなり, base環境が自動で有効化されなくなる.

もし, 再び自動有効化をオンにしたい場合は, falseをtrueに変更して同じコマンドを実行すればよい. 

conda config --set auto_activate_base true

手動で(base)環境を有効/無効にする

(base)環境を手動で有効にする

conda activate base

このコマンドを実行すると, 現在のターミナルセッションでbase環境が有効になり, プロンプトの先頭に(base)が表示される. Anacondaのルート環境にあるパッケージ（Python, Jupyterなど）を利用したい場合に実行する.

(base)環境を手動で無効にする

conda deactivate

このコマンドを実行すると, (base)の表示が消え, base環境が無効化される. これで通常の環境に戻って来れる.
